구글에서 검색했을 때 별점, FAQ, 날짜가 표시되는 결과를 본 적 있으신가요? 그게 바로 구조화 데이터의 힘입니다. 구조화 데이터와 AEO 전략을 함께 적용하면, 일반 검색은 물론 AI 답변 박스에도 노출될 수 있습니다.

> 핵심 답변: 구조화 데이터는 검색엔진이 콘텐츠를 정확히 이해하게 해주는 JSON-LD 마크업입니다. Article, FAQPage, WebSite 3가지를 적용하면 구글 리치 결과 노출 확률이 2.7배 높아집니다 (출처: Schema.org 공식 보고, 2026).

## 구조화 데이터란? 왜 중요한가요?

구조화 데이터는 검색엔진과 AI에게 콘텐츠의 의미를 알려주는 코드입니다. 사람이 "이건 블로그 글이고, 작성자는 누구고, 언제 쓴 거야"라고 이해하는 것처럼, 기계도 같은 정보를 읽을 수 있게 해줍니다.

### 구조화 데이터의 효과

| 효과 | 수치 |
|------|------|
| 리치 결과 노출 | CTR +30-50% 증가 |
| 구글 AI Overview 인용 | 구조화 데이터 있는 페이지 우선 선택 |
| 네이버 AI 요약 | 명확한 구조의 콘텐츠 우대 |
| 음성 검색 답변 | FAQ 스키마가 음성 검색 답변에 활용 |

> 핵심: 구글은 2026년 기준 약 35%의 검색 결과에 리치 스니펫을 표시합니다. 구조화 데이터 없이는 이 영역에 노출될 수 없습니다 (출처: Google Search Central, 2026).

## 필수 구조화 데이터 3가지

### 1. WebSite 스키마 (사이트 전체)

`app/layout.tsx`에 한 번만 추가하면 됩니다. 사이트 전체를 대표하는 정보를 담습니다.

```json
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "AI 브리핑",
  "url": "https://seroai.xyz",
  "description": "AI 뉴스, 정부사업, AI 도구 가이드",
  "inLanguage": "ko-KR"
}
```

### 2. Article 스키마 (각 블로그 포스트)

각 포스트 페이지의 `<head>`에 추가합니다. 검색엔진이 글의 제목, 작성자, 발행일을 정확히 인식하게 해줍니다.

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "포스트 제목",
  "description": "포스트 요약",
  "author": {
    "@type": "Organization",
    "name": "AI 브리핑"
  },
  "datePublished": "2026-03-30",
  "dateModified": "2026-03-30",
  "publisher": {
    "@type": "Organization",
    "name": "AI 브리핑",
    "url": "https://seroai.xyz"
  }
}
```

### 3. FAQPage 스키마 (AEO 핵심!)

FAQ 섹션이 있는 페이지에 추가합니다. 구글 검색에서 FAQ 리치 결과로 노출되며, AI 답변 박스의 핵심 소스가 됩니다.

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Claude AI는 무료로 사용할 수 있나요?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "네, Claude AI는 무료 플랜에서 하루 30회 사용 가능합니다."
      }
    }
  ]
}
```

> 핵심 답변: 블로그에 필수로 적용할 구조화 데이터는 WebSite(사이트 전체), Article(각 포스트), FAQPage(FAQ 있는 페이지) 3가지입니다. JSON-LD 형식으로 `<script type="application/ld+json">`에 넣으면 됩니다.

## AEO 콘텐츠 전략 — AI 답변 박스 노출법

구조화 데이터가 "기술적 신호"라면, AEO 콘텐츠 전략은 "콘텐츠 작성 방식"을 바꾸는 것입니다. 구글 AI Overview와 네이버 AI 요약은 질문 → 즉답 구조의 콘텐츠를 선호합니다.

### BAD vs GOOD 비교

| 규칙 | BAD | GOOD |
|------|-----|------|
| 제목 | Claude AI 2026년 완벽 활용법 | Claude AI 어떻게 사용하나요? 2026년 활용법 7가지 |
| 첫 문단 | 오늘은 Claude AI에 대해 알아보겠습니다... | Claude AI는 Anthropic이 개발한 AI로, 무료 플랜에서 하루 30회 사용 가능합니다. |
| 소제목 | ## 기능 소개 | ## Claude AI의 주요 기능은 무엇인가요? |
| FAQ | (없음) | ## 자주 묻는 질문 + Q&A 2-3개 |

> 꿀팁: 제목에 "어떻게", "무엇", "왜", "언제", "얼마" 같은 의문사를 넣으면 AI 답변 박스 노출 확률이 약 2배 높아집니다. 단, 제목이 자연스러워야 합니다.

### AEO 핵심 4원칙

1. 질문형 제목 — 사용자가 실제로 검색할 법한 질문 포함
2. 첫 문단 즉답 — 50-100자로 핵심 답변 (AI가 바로 추출 가능)
3. 질문형 소제목 — H2/H3에도 질문 키워드 변형 배치
4. FAQ 섹션 — 글 하단에 자주 묻는 질문 2-3개 + Schema.org FAQPage 마크업

## 구조화 데이터 테스트 방법

작성한 구조화 데이터가 올바른지 확인하는 도구:

1. Google Rich Results Test — 리치 결과 미리보기 가능
2. Schema.org Validator — JSON-LD 문법 검증
3. Google Search Console — "개선사항" 탭에서 오류 확인

> 핵심: 구조화 데이터를 추가한 후 반드시 Google Rich Results Test로 검증하세요. 문법 오류가 하나라도 있으면 전체 스키마가 무시됩니다.

## 자주 묻는 질문

### Q1. 구조화 데이터를 넣으면 바로 리치 결과가 나오나요?
A1. 아닙니다. 구조화 데이터는 리치 결과의 필요 조건이지 충분 조건은 아닙니다. 구글이 콘텐츠 품질을 평가한 후 표시 여부를 결정하며, 적용 후 2-4주 뒤에 노출되기 시작합니다.

### Q2. FAQPage 스키마는 모든 글에 넣어야 하나요?
A2. FAQ 섹션이 있는 글에만 넣으세요. 실제 질문과 답변이 없는데 스키마만 넣으면 구글이 스팸으로 판단할 수 있습니다. 진짜 유용한 Q&A가 있을 때만 사용하세요.

### Q3. Next.js에서 구조화 데이터 추가하는 가장 좋은 방법은?
A3. `<script type="application/ld+json">`을 JSX에서 `dangerouslySetInnerHTML`로 삽입하는 방법이 가장 표준적입니다. Next.js의 `metadata` API에서는 아직 JSON-LD를 직접 지원하지 않습니다.
