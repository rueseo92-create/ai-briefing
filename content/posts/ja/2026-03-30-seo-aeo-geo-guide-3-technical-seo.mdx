サーチコンソールに登録したら、次は検索エンジンとAIクローラーがサイトを正しく読み取れる環境を作る必要があります。この記事では技術的SEOの核心ファイル4つを扱います。

> 核心回答：技術的SEOファイルは、robots.txt（クローリング規則）、sitemap.xml（ページ一覧）、feed.xml（RSSフィード）、llms.txt（AIクローラー用要約）の4つです。Next.jsではすべて自動または半自動で生成できます。

---

## robots.txt — クローリング許可設定

robots.txtは検索エンジンクローラーに「どこを読んでもよく、どこはだめなのか」を知らせるファイルです。2026年にはAIクローラー（GPTBot、Claude-Webなど）も考慮する必要があります。

### 推奨robots.txt設定

```
User-agent: *
Allow: /

# AIクローラーも許可
User-agent: GPTBot
Allow: /
User-agent: ChatGPT-User
Allow: /
User-agent: Claude-Web
Allow: /
User-agent: PerplexityBot
Allow: /

Sitemap: https://seroai.xyz/sitemap.xml
```

> コツ：AIクローラーをブロックすると、ChatGPT、Perplexityなどでブログコンテンツが引用されなくなります。GEOのために必ず許可しておきましょう。

### 主要AIクローラー一覧

| クローラー | サービス | 推奨設定 |
|--------|--------|-----------|
| GPTBot | ChatGPT/OpenAI | Allow |
| ChatGPT-User | ChatGPTブラウジング | Allow |
| Claude-Web | Claude/Anthropic | Allow |
| PerplexityBot | Perplexity AI | Allow |
| Googlebot | Google検索 | Allow |
| Yeti | Naver検索 | Allow |

## sitemap.xml — 全ページ一覧提供

サイトマップは検索エンジンにブログのすべてのページURLと更新日を知らせるファイルです。Googleの公式文書によると、サイトマップを提出したサイトはインデックス速度が平均4倍速くなります（出典：Google Search Central、2026）。

### Next.jsで自動生成する

```typescript
// app/sitemap.ts
export default async function sitemap() {
  const posts = getAllPosts(); // すべての投稿を取得
  return [
    { url: 'https://seroai.xyz', lastModified: new Date() },
    { url: 'https://seroai.xyz/posts', lastModified: new Date() },
    ...posts.map(post => ({
      url: `https://seroai.xyz/posts/${post.slug}`,
      lastModified: new Date(post.date),
    })),
  ];
}
```

> 核心：Next.js App Routerで`app/sitemap.ts`を作成すると、ビルド時に自動でsitemap.xmlが生成されます。新しい記事を公開してデプロイすれば自動更新されるため、別途管理が不要です。

## feed.xml (RSS) — 購読およびNaver提出用

RSSフィードはブログ記事を購読できる標準形式です。Naverサーチアドバイザーにも必須提出項目でもあります。RSSを提出するとNaverが新しい記事をより早く認識します。

### RSS生成方法

```bash
npm install feed
```

> 核心回答：feed.xmlはNaverサーチアドバイザーに必須提出すべきファイルです。`feed`パッケージで自動生成し、Next.js Route Handler（`app/feed.xml/route.ts`）で提供すればよいです。

## llms.txt — AIクローラーのためのサイト要約（GEO核心）

llms.txtは2025年に提案された新しい標準で、AIクローラーにサイトの核心情報を要約して知らせるファイルです。robots.txtが「どこを読め」なら、llms.txtは「我々のサイトはこんな所だ」と説明する役割です。

### 推奨llms.txt構造

```markdown
# サイト名
> 一行紹介

## 主要コンテンツ
- カテゴリ1：説明
- カテゴリ2：説明

## サイト情報
- URL: https://seroai.xyz
- 言語：日本語
- 更新周期：毎日
- RSS: /feed.xml
- サイトマップ: /sitemap.xml
```

> 核心：llms.txtは`public/llms.txt`にデプロイすればよいです。AIクローラーがこのファイルを読んでサイトをより理解するようになり、AI回答で引用される確率が高まります。

| ファイル | 位置 | 用途 | 必須度 |

## よくある質問

### Q1. robots.txtでAIクローラーをブロックするとどうなりますか？
A1. ChatGPT、PerplexityなどのAIサービスでブログコンテンツが引用されなくなります。トラフィック流入の機会を失うことになるため、GEOを望むなら必ず許可する必要があります。

### Q2. llms.txtは必ず作成する必要がありますか？
A2. 必須ではありませんが、強く推奨します。AIクローラーがサイト構造をより理解するようになり、関連質問への回答で引用される可能性が高まります。作成には5分で十分です。

### Q3. sitemap.xmlにすべてのページを入れる必要がありますか？
A3. 検索に露出させたいページのみ入れればよいです。404ページ、管理者ページなどは除外してください。Next.jsの自動生成を使用すると、`app`ディレクトリの公開ページのみ自動で含まれます。
