Si ya te has registrado en Search Console, ahora necesitas crear un entorno donde los motores de búsqueda y crawlers de IA puedan leer tu sitio correctamente. En este artículo cubrimos los 4 archivos esenciales del SEO técnico.

> Respuesta clave: Los archivos SEO técnicos son 4: robots.txt (reglas de crawling), sitemap.xml (lista de páginas), feed.xml (feed RSS), llms.txt (resumen para crawlers de IA). En Next.js se pueden generar todos de forma automática o semiautomática.

---

## robots.txt — Configuración de permisos de crawling

robots.txt es un archivo que le dice a los crawlers de motores de búsqueda "dónde pueden leer y dónde no". En 2026, también debemos considerar los crawlers de IA (GPTBot, Claude-Web, etc.).

### Configuración recomendada de robots.txt

```
User-agent: *
Allow: /

# También permitir crawlers de IA
User-agent: GPTBot
Allow: /
User-agent: ChatGPT-User
Allow: /
User-agent: Claude-Web
Allow: /
User-agent: PerplexityBot
Allow: /

Sitemap: https://seroai.xyz/sitemap.xml
```

> Consejo: Si bloqueas los crawlers de IA, el contenido de tu blog no será citado en ChatGPT, Perplexity, etc. Para GEO, asegúrate de permitirlos.

### Lista de principales crawlers de IA

| Crawler | Servicio | Configuración recomendada |
|---------|----------|---------------------------|
| GPTBot | ChatGPT/OpenAI | Allow |
| ChatGPT-User | Navegación ChatGPT | Allow |
| Claude-Web | Claude/Anthropic | Allow |
| PerplexityBot | Perplexity AI | Allow |
| Googlebot | Búsqueda Google | Allow |
| Yeti | Búsqueda Naver | Allow |

## sitemap.xml — Proporcionar lista completa de páginas

El sitemap es un archivo que informa a los motores de búsqueda sobre todas las URLs de las páginas del blog y sus fechas de actualización. Según la documentación oficial de Google, los sitios que envían sitemaps tienen una velocidad de indexación promedio 4 veces más rápida (fuente: Google Search Central, 2026).

### Generación automática en Next.js

```typescript
// app/sitemap.ts
export default async function sitemap() {
  const posts = getAllPosts(); // Obtener todos los posts
  return [
    { url: 'https://seroai.xyz', lastModified: new Date() },
    { url: 'https://seroai.xyz/posts', lastModified: new Date() },
    ...posts.map(post => ({
      url: `https://seroai.xyz/posts/${post.slug}`,
      lastModified: new Date(post.date),
    })),
  ];
}
```

> Clave: En Next.js App Router, crear `app/sitemap.ts` genera automáticamente sitemap.xml durante el build. Se actualiza automáticamente cuando publicas y despliegas nuevos artículos, sin necesidad de gestión adicional.

## feed.xml (RSS) — Para suscripciones y envío a Naver

El feed RSS es un formato estándar para suscribirse a artículos del blog. También es un elemento obligatorio para enviar a Naver Search Advisor. Enviar RSS hace que Naver reconozca nuevos artículos más rápidamente.

### Método de generación RSS

```bash
npm install feed
```

> Respuesta clave: feed.xml es un archivo que debe enviarse obligatoriamente a Naver Search Advisor. Se puede generar automáticamente con el paquete `feed` y servir desde Next.js Route Handler (`app/feed.xml/route.ts`).

---

## llms.txt — Resumen del sitio para crawlers de IA (núcleo GEO)

llms.txt es un nuevo estándar propuesto en 2025 que proporciona a los crawlers de IA un resumen de la información clave del sitio. Si robots.txt dice "dónde leer", llms.txt explica "nuestro sitio es así".

### Estructura recomendada de llms.txt

```markdown
# Nombre del sitio
> Descripción de una línea

## Contenido principal
- Categoría 1: descripción
- Categoría 2: descripción

## Información del sitio
- URL: https://seroai.xyz
- Idioma: coreano
- Frecuencia de actualización: diaria
- RSS: /feed.xml
- Sitemap: /sitemap.xml
```

> Clave: llms.txt se puede desplegar en `public/llms.txt`. Los crawlers de IA leen este archivo y comprenden mejor el sitio, aumentando la probabilidad de ser citado en respuestas de IA.

## Preguntas frecuentes

### P1. ¿Qué pasa si bloqueo los crawlers de IA en robots.txt?
R1. El contenido del blog no será citado en servicios de IA como ChatGPT, Perplexity, etc. Perderás oportunidades de tráfico, así que si quieres GEO, debes permitirlos obligatoriamente.

### P2. ¿Es necesario crear llms.txt?
R2. No es obligatorio pero sí muy recomendado. Los crawlers de IA comprenderán mejor la estructura del sitio, aumentando la posibilidad de ser citado en respuestas a preguntas relacionadas. Solo toma 5 minutos crearlo.

### P3. ¿Debo incluir todas las páginas en sitemap.xml?
R3. Solo incluye las páginas que quieres que aparezcan en búsquedas. Excluye páginas 404, páginas de administración, etc. Usar la generación automática de Next.js incluye automáticamente solo las páginas públicas del directorio `app`.
